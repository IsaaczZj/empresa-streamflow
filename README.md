# empresa-streamflow

## Setor
Streaming

## Contexto do problema
A StreamFlow opera uma plataforma com alto volume de tráfego e eventos ao vivo. Em incidentes operacionais, os times executam procedimentos diferentes por falta de runbook único, causando aumento de tempo de recuperação e inconsistência nas comunicações.

## Objetivo do desafio
Produzir um runbook operacional padronizado para resposta a incidentes, diagnóstico inicial, mitigação, escalonamento e pós-incidente.

## Instruções resumidas para os grupos
1. Faça um fork deste repositório para a conta do grupo.
2. Trabalhe exclusivamente no fork do grupo.
3. Preencha a documentação na pasta `docs/`, usando `docs/runbook-template.md` como referência e consolidando o resultado em `docs/runbook.md`.
4. Entregue o link do fork do grupo conforme orientação da disciplina.
